1/19/2016

We are moving to an electronic and cloud-based lab notebook. Zach found a solution that uses Google Drive for storage, StackEdit for editing, and GitHub for publishing. Zach has the framework in place.

As a new user getting started, the procedure goes like this:

  1. This assumes that you have logged out of Google and GitHub, and have cleared browser cookies.
  2. Navigate to stackedit.io
  3. (Optional) In the left pane window, select all and delete to create a clean document. Insert some custom text (e.g. “Hello World!”).
  4. In the upper right is a file name (e.g. “Hello!”); click on it to select it; delete it and replace it with a custom name (e.g. “StackEditTest”).
  5. # Syncronize Save on Google Drive
    -This will save “StackEditTest” to your Google Drive account and keep it syncronized. Click OK
    -Redirection: You are being redirected to Google authorization page. Click OK
  6. Log in to Google with lab credentials.
    -“StackEditTest” is now synced to Google Drive.
  7. # Publish GitHub
    -Repository: LettSpinorLab/LabNotebook.git
    -Branch: gh-pages
    -File path: StackEditTest.html
    -Format: Template
    -Redirection: You are being redirected to GitHub authorization page. Click OK
  8. Log in to GitHub with lab credentials.
    -“StackEditTest.html” is now published on GitHub.
  9. To view rendered doc, go to: LettSpinorLab.github.io/LabNotebook/StackEditTest.html

If a lab notebook file has already been created, synced, and published using the above steps, but you are using a new computer, follow these steps:

  1. Navigate to stackedit.io
  2. # Syncronize Open from Google Drive
    -Redirection: You are being redirected to Google authorization page. Click OK
  3. Log in to Google with lab credentials.
  4. Select the desired document.
  5. Edit document.

When working with a lab notebook in a browser that is logged in to the lab Drive and GitHub accounts, in StackEdit:
-To manual sync to Drive: Sync button (upper right)
-To publish to GitHub: Upload/Push button (upper right)

A list of useful StackEdit symbols/icons can be found here:
https://stackedit.io/res/libs/fontello/demo.html

To add an extension (e.g. timestamp)
# Settings ExtensionsUserCustom extension
and insert it in the JavaScript code box.

23:49:29-20/01/2016

Added an EOT optical isolator after the TA. Might need to be optimized. The output of the TA is rather eccentric, with major axis vertical to table. Ideally, we would reshape with an anamorphic prism pair. The prism pair that was on the table has a chip in it that makes it unusable. (Happened during move?) Without one, we’ll lose even more light through a spatial filter. Will leave space for a prism pair, and do a “dry run” of setting up spatial filter.

To start, getting a sense of beam size after TA. Doing a 10/90 knife edge measurement to get rough horizontal (minor) axis diameter.

Settings:
seed laser = 69 mW @ 1064.660 nm, nominally 30 mW into TA
TA current = 809.5 mA
power after EOT isolator = 100 mW
90% (90 mW) micrometer reading = 0.2738”
10% (10 mW) micrometer reading = 0.3082”
10/90 width = 0.0344 in = 8.7376E-4 m (or about 1 mm)

This article gives an overview of aligning a spatial filter. In particular, it suggests what objective/pinhole combination to use depending on wavelength and beam diameter. The table given is for 632 nm light with a beam diameter of 1 mm. We have a similar beam diameter, but our wavelength is about 60% longer. For a 20x magnification objective, we would want to use a pinhole with a diameter of about (1.67 x 15 microns) = 25 microns.
